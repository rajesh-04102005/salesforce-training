# Day 2 - Questions and Answers

## 1. What is an App in Salesforce?

### Answer:
An App in Salesforce is a collection of related tabs, objects, and tools grouped together for a specific business purpose.

Examples:
- Sales App
- Service App
- College Management App

Apps help users access all required features in one place.

---

## 2. What is an Object?

### Answer:
An Object in Salesforce is similar to a database table used to store records and data.

Objects contain:
- Fields (columns)
- Records (rows)

Examples:
- Account
- Contact
- Student
- Course

---

## 3. Difference between App and Object

### Answer:

| App | Object |
|---|---|
| Collection of tabs and tools | Stores data |
| Organizes business functionality | Works like a database table |
| Used for navigation and UI | Contains fields and records |

---

## 4. What is multi-tenant architecture? (basic understanding)

### Answer:
Multi-tenant architecture means multiple organizations share the same Salesforce infrastructure while keeping their data secure and isolated.

Benefits:
- Lower cost
- Automatic updates
- Scalability
- High availability

---

## 5. When should we use configuration instead of code?

### Answer:
Configuration should be used when requirements can be solved using built-in no-code or low-code tools.

Examples:
- Creating custom objects
- Building workflows using Flow Builder
- Creating validation rules
- Customizing page layouts

Configuration is faster, easier to maintain, and does not require programming.

---

## 6. How does Salesforce allow developers to extend functionality?

### Answer:
Salesforce allows developers to extend functionality using:

- Apex programming language
- Lightning Web Components (LWC)
- APIs
- Integrations
- AppExchange apps

Examples:
- Payment gateway integration
- Custom approval systems
- External application integration
- Advanced automation logic
